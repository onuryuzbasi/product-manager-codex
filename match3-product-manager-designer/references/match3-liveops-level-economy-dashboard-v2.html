Why this metric

  • Gercek sifir nadir olabilir; pratikte oyuncu continue alamadigi anda sikisiyor.
  • Bu nedenle `hard zero` yerine `functional zero` izleniyor.
  • 87-89 ve 94-95 arasi belirgin risk bolgesi.

Counting rule

  • balance >= 900 iken balance < 900 olursa 1 entry say.
  • Threshold altindayken tekrar tekrar sayma.
  • Tekrar 900 ustune cikip yeniden altina inerse yeni entry say.

Data Definition

V1 icin gereken minimum veri alanlari.

Minimum fields

  • level_id
  • user_id veya aggregate user count
  • coin_balance_after_level
  • continue_price
  • spend_happened
  • gift_coin_amount
  • reward_coin_amount

Core event rule

  • below_continue_threshold_entered event'i state change bazli uretilmeli.
  • Threshold altinda kaldigi surece yeni event yazilmamali.
  • Threshold ustune cikip tekrar altina inerse yeni event yazilmali.
