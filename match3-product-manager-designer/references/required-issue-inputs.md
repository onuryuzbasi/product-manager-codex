# Minimum Inputs For Issue Planning

Ask only these if missing:

1. Issue type: feature, improvement, bugfix, event, or experiment
2. Target player outcome: what should improve for the player
3. Business/product objective: retention, conversion, progression, stability, etc.
4. Affected area: level flow, core board, booster, economy, UI, liveops, onboarding
5. Scope constraints: deadline, platform, dependencies, blocked teams

If unavailable, continue with explicit assumptions and note validation needs.
