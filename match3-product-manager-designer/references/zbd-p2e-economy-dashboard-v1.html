ZBD P2E Economy Dashboard v1

Bu dashboard, P2E sisteminin karli sekilde yonetilmesi icin tek ekranda okunacak ekonomi kontrol panelidir. Mantik su: kullanicidan gelen gelir, kullaniciya verilen reward ve bonus maliyetinden yuksek kalmali; sistem sadece calismamali, net katkı uretmelidir.

Core Question: Kullanicidan kazandigimizdan fazlasini veriyor muyuz? Decision Unit: Country + Bucket + Level Band + Config Snapshot Main Rule: Revenue > Reward Cost

Core Formulas

Dashboard ve canli tuning butun kararlari bu hesaplar uzerinden verecek.

Unit Economics

total_revenue = rewarded_revenue + inter_revenue + iap_revenue total_reward_cost = level_reward_cost + welcome_bonus_cost + welcome_back_cost + promo_bonus_cost + cashout_cost net_contribution_margin = total_revenue - total_reward_cost reward_cost_ratio = total_reward_cost / total_revenue

Budget Rules

allowed_reward_budget = total_revenue * target_cost_ratio safe_reward_budget = (total_revenue * target_cost_ratio) - reserve_buffer reward_velocity = reward_cost_per_user_per_day break_even_day = first day where cumulative_revenue exceeds cumulative_reward_cost

Reward Budget Rules

Rule 1: Total reward cost hicbir zaman total revenue'ya yaklasmamalı; sistemin net margin birakmasi zorunlu.
Rule 2: Welcome bonus, welcome back ve promo gift birer growth tool'dur; herkese default entitlement gibi dagitilmamali.
Rule 3: Reward tuning yaparken ayni anda sadece bir knob grubu degismeli. Aksi halde sonuc karisir.
Rule 4: Her config degisikligi `config_snapshot_id` ile izlenmeli ve revenue/cost sonucu ayrik okunmali.
Rule 5: Reward cost ratio warning zone'a girerse once bonuslar, sonra level reward pacing, sonra ad multiplier ayarlanir.
